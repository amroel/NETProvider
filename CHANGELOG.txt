2004-08-04 Carlos Guzman Alvarez  <carlosga@telefonica.net>

	* Backported the Database schema support changes from v1.7
    
        This includes:

        * New FbConnection methods

            - GetChema()
            - GetSchema(string)
            - GetSchema(string[])

        * Change the names of the FbDbSchemaType elements to remove the underscore.

        * Update all the classes in the FirebirdSql.Data.Firebird/DbSchema directory.

        * Update the implementation of the FbCommandBuilder.DeriveParameters static method.

        * Update the TypeHelper and FbParameter classes to implement the new data type helper methods.

        Some of the changes in the Schema Results are:

	        - Data types schema -

        * New schema that will return the data types information.

	        - Metadata collections -

        * New schema that will return all the supported schemas, and the number of restrictions of each one.
	
        	- Columns Schema -

        * The COLUMN_DATA_TYPE now holds the datatype name.

        * New columns:
           
	        - IS_ARRAY

        * Removed columns:

	        - PROVIDER_TYPE (to known that we can search in the DataTypes schema)
        	- PRIMARY_KEY
	        - UNIQUE_KEY

        The reason is that the query with that columsn gets a lot time to be executed in complex database (with a lot of tables, columns, etc...)

        - Tables schema -

	        * The IS_SYSTEM_TABLE column now will have 0/1 as values

		* Not it's no possible to filter by the IS_SYSTEM_TABLE column,
		instead filter by TABLT_TYPE can be done using:

			- TABLE
			- VIEW
			- SYSTEM TABLE

        - Generators schema -

	        * The IS_SYSTEM_GENERATOR column now will have 0/1 as values

        - Functions schema -

	        * The IS_SYSTEM_FUNCTION column now will have 0/1 as values

        - Triggers schema -

	        * The IS_SYSTEM_TRIGGER column now will have 0/1 as values

        - Primary keys schema -

	        * Now it's possible to filter by COLUMN_NAME

        - Unique keys schema -

	        * Now it's possible to filter by COLUMN_NAME
